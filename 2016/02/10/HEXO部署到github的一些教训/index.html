HEXO部署到github的一些教训

2016/02/10
  • deploy
  • hexo

这是摘要

为了用hexo deploy这个命令来部署网页，真的走了好多弯路！

简述

HEXO生成的静态网页存储在public文件夹中，在运行deploy命令时实际上是把public里面的内容复制到了deploy目录然后push到设置好的github仓库中。
道理都明白，可是部署时就报错了，然后很傻逼的看到网上说要用这个命令

如果想要正常发布网页，千万不要用这个命令！！
这样虽然不会出错，但是会把整个源文件全部替换到远程仓库中，然后网页的位置变了看不到了呀
这不是我想要的结果！
那么，报错的正确解决方式应该是这样的：

错误1，fatal: Not a git repository (or any of the parent directories): .git
解决方法：删除deploy文件夹！（？）

错误2，fatal: could not read Username for ‘https://github.com'
解决方法：将https部署改为ssh部署
参考链接

然后给一个觉得还不错的搭建Hexo的教程链接

另外，发现可以不用安装git，换成github desktop,这样貌似可以不用手动设置ssh，在程序中登陆之后直接用git shell来执行部署操作。
